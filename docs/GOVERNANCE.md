# Governance

Nazar is, for now, a **maintainer-led** project. The maintainer is Alex Radunet.

## Decisions
- **Day-to-day** technical decisions: the maintainer, guided by the project values.
- **Direction** (architecture, scope, licensing): discussed in issues; the maintainer
  decides, in the open.
- **Values that arbitrate trade-offs:** sovereignty, open standards, privacy by
  default, no lock-in, and KISS / YAGNI / suckless.

## Becoming a contributor
Open issues, send focused PRs, and sign the CLA (see [CONTRIBUTING.md](./CONTRIBUTING.md)).
Consistent, quality contributions earn triage/commit rights over time.

## Roadmap
Maintained in the open. The core stays narrow on purpose; new components are pulled in
only when actually needed (YAGNI).

## Sustainability
Funded by (a) public-interest grants for the FOSS core (e.g. NGI Zero) and (b) an
optional commercial managed-hosting tier. See [OPEN_CORE_BOUNDARY.md](./OPEN_CORE_BOUNDARY.md).
