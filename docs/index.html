Nazar

The studio for your life.

About

An OS-agnostic Pi.Dev extension suite for building a personal AI assistant that knows what you keep, remembers what matters, and helps you orchestrate your life.

Memory, voice, and your knowledge base — woven together as a single personal world. Runs on Linux, Windows, macOS. Stores everything in a portable Obsidian vault you own. Local-first by default. Networked integrations are deferred until they are essential.

Quick start ↓

Principles

  • OS-agnostic. No NixOS, no Docker host assumptions. Install dependencies through the package manager of your platform. The extensions stay portable.
  • Local-first. Your memory, transcripts, and state live on your machine in a portable Obsidian vault. You can move it. You can read it without Nazar.
  • Pi-native. Built on the Pi.Dev coding-agent extension API. The agent is a first-class part of your environment, not a chatbot bolted on the side.
  • Privacy-first. Private memory, runtime credentials, voice models, and runtime state stay out of git by default.
  • KISS & inspectable. TypeScript modules you can read in an afternoon. No magical build steps. No hidden services.

Quick start

Step 1

Install Pi.Dev

Nazar runs as a set of extensions inside the Pi.Dev coding agent. Install Pi first per its docs.

Step 2

Install Nazar

pi install @nazar/nazar-pi

This registers the core, memory, and voice extensions in your Pi settings and pulls the optional local voice dependency when enabled.

Step 3

Point Nazar at your memory vault

export NAZAR_HOME="$HOME/NazarVault"

Then run setup inside the agent:

/nazar-setup memory

Nazar will scaffold a PARA-style vault structure if one doesn't already exist.

Step 4

Verify

/nazar-status

You should see green checks for memory and voice (if configured).

Architecture

Nazar is opinionated about where things live but agnostic about how the host machine is configured.

  • Pi.Dev is the runtime. Extensions register with the Pi extension API. No separate daemon, no service to install.
  • Your Obsidian vault is the database. Personal memory lives in your vault. The 05_Nazar/ directory is the AI/system control plane.
  • Extensions are TypeScript modules. Each is a single .ts file plus an optional sub-directory. Edit .pi/settings.json to add or remove.
  • Host setup is your problem. Install Node.js, audio helpers, GitHub CLI etc. through apt / dnf / brew / winget / nix-env.

Vault structure (scaffolded by /nazar-setup)

NazarVault/ ├── 00_Inbox/ — shared human/AI capture ├── 01_Projects/ — human-owned ├── 02_Areas/ — human-owned ├── 03_Resources/ — human-owned ├── 04_Archive/ — cold storage (excluded from search) └── 05_Nazar/ — AI / system control plane ├── llm-wiki/ │ ├── raw/ — AI's raw notes │ └── wiki/ — Karpathy-style compiled wiki ├── runtime/ — generated state, rollups, journals ├── ai-workbench/ — proposals, drafts, scratch └── operator-log/ — audit log

Privacy boundary

  • In git: code, tests, docs, templates, public AI/infrastructure memory.
  • Out of git (always): private memory pages, generated rollups, journals, source reports, runtime credentials, local voice models, your personal Obsidian vault.

Do not commit secrets. Do not commit raw session transcripts. Do not expose SSH or RDP to the internet without an explicit VPN/tunnel plan.

Status

Nazar is in active development. The core memory/voice extensions work end-to-end on Linux, Windows, and macOS. The Obsidian-backed memory layer is the current focus.

Roadmap items include richer multi-vault support and a managed-installer experience for non-developers.
