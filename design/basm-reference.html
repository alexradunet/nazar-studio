// 02 — Components

Visual building blocks

All components are pixel-hard: 2px borders, 3px radius, hard offset shadows, no rounded blobs or soft gradients.

Woven dividers

folk-band — use between major narrative sections
stitch — subtle separator inside sections

Buttons — pixel-hard, uppercase mono, hard shadow

Cards — gold corner notch, 2px border, lift on hover

⌂

Your data, your box

Plain Markdown in a vault you own — never the price of admission to your own memory.

☉

Local model first

The terminal defaults to the local model. Personal context stays on the box.

☊

Skills grow as eyes

Each approved procedure becomes a new eye watching one more domain of your life.

Files over platforms

Suggest → approve

// 03 — Terminal UI

RPG dialog panels

The Pi terminal is Nazar's owned daily surface — an old-school fantasy RPG command interface that is compact, truthful, and useful. Panels use ANSI-colored RPG borders, fixed avatar cells, and colored role identities.

// 04 — The Eyes

Design rules for any Nazar surface

✓
Companion-first copy. Lead with the wise companion (help, memory, growth). Privacy / local-first is the quiet second beat.
✓
Woven, not rendered. Pixel-hard edges, 2px borders, hard shadows. No glassmorphism, no AI gradients, no rounded blobs.
✓
Plain technical truth. Only describe what's shipped. The honesty ledger is §7 of design/identity.md.
✓
One voice. Heads are flavor, not separate agents. The terminal voice is always Nazar's — warm, wise, never hypey.
✗
Don't imply separate AIs. Heads are domains, not agents. Don't write "activate the Keeper" or "Keeper replies."
✗
No hoard/vault/treasure copy. Nazar is a companion who protects — not a charm that trades on fear of the evil eye. Lead with help.
